Does Hexo markdown escapes HTML special characters?

2023-02-01

In Hexo, A markdown-format post is converted to HTML.
HTML document needs to escape special characters like ‘Less than’, ‘Greater than’ and ‘Apostropy’ as:
‘<’, ‘>’,’&apos; / &#x27;’, ‘&rarr;’.

Result of entities:

  • < and > must escape manually markdown writer himself / herself.
  • Other HTML entities like aportrophe(&apos;) are no need to escape.

Escaping special characters ‘LT’(<) and ‘GT’(>)

  • Two special characters:[ less-than (<) and _greater-than_ (>) ] need forwarding backslash inserted as:[“\<”, “\>”] not to be treated as embedding HTML tags.
